# 深度分析技能使用说明

## 概述

你现在拥有一个强大的**深度分析技能**，它整合了五大思维工具：
1. 第一性原理
2. 系统思维（冰山模型）
3. 逆向思维
4. 多元思维格栅
5. 批判思考（Paul-Elder框架）

这个技能已经成功应用于"美伊冲突系统性分析"，生成了一份完整的分析报告。

## 文件位置

1. **分析报告示例**：`~/美伊冲突系统性分析报告.md`
2. **全局技能文件**：`~/.claude/skills/deep-analysis.md`
3. **完整框架文档**：`~/.claude/skills/integrated-thinking-framework.md`
4. **本使用说明**：`~/深度分析技能使用说明.md`

## 如何使用

### 方式1：直接触发

在对话中说：
- "帮我深度分析..."
- "用系统性思维分析..."
- "我需要全面分析..."

### 方式2：控制分析深度

- **快速模式**："快速分析这个问题"（5-10分钟）
- **标准模式**："帮我分析..."（20-30分钟，默认）
- **深度模式**："深度研究这个问题"（30-60分钟）

### 方式3：指定思维工具

- "用第一性原理分析..."
- "用系统思维的冰山模型看看..."
- "用逆向思维分析..."
- "从多元视角分析..."

## 工作流程

### 第1步：对话式澄清
我会问你5个问题：
1. 具体发生了什么？（事实层）
2. 你认为原因是什么？（假设层）
3. 理想状态是什么？（目标层）
4. 有哪些限制条件？（约束层）
5. 其他人会怎么看？（视角层）

你只需简短回答，或者说"全面分析"跳过这一步。

### 第2步：系统性研究
我会并行启动3个研究代理：
- 代理1：历史根源和结构性原因
- 代理2：影响机制和传导路径
- 代理3：应对策略和行动建议

你会看到实时的研究进度更新。

### 第3步：结构化输出
我会生成一份完整的 Markdown 分析报告，包含：
- 执行摘要（3句话核心洞察）
- 8个完整章节的深度分析
- 可执行的行动建议
- 风险地图和验证方法

## 输出示例

参考 `~/美伊冲突系统性分析报告.md`，你会得到：

1. **执行摘要**：3句话总结核心洞察
2. **问题拆解**：假设清单、基本事实、逻辑推导
3. **系统性分析**：冰山模型四层分析
4. **多元视角**：从4-5个学科交叉分析
5. **具体影响**：对不同人群的具体影响
6. **行动建议**：按优先级和风险等级分类
7. **态势评估**：短期、中期、长期展望
8. **风险地图**：高概率风险、高影响风险、缓解策略

## 适用场景

### ✅ 适合使用的场景

- **复杂问题**：涉及多个维度、多方利益相关者
- **战略决策**：需要全面评估风险和机会
- **系统性问题**：问题反复出现，需要找根本原因
- **跨学科问题**：需要从多个专业视角分析

### ❌ 不适合使用的场景

- **简单问题**：答案明确，不需要深度分析
- **技术问题**：纯技术实现，不涉及复杂决策
- **紧急问题**：需要立即行动，没时间深度分析

## 实战案例

### 案例1：美伊冲突分析（已完成）

**问题**：为什么美伊战争总是不能结束？对世界金融秩序有何影响？普通人应该怎么办？

**使用工具**：
- 第一性原理：拆解到身份对立、地缘博弈、能源体制竞争
- 系统思维：分析冲突的自我强化循环
- 多元视角：地缘政治学、经济学、金融学、历史学
- 逆向思维：最坏情况分析

**输出**：12,000字完整分析报告，包含具体的应对策略

### 案例2：职业发展决策（模板）

**问题**：是否应该换工作？

**分析框架**：
1. 第一性原理：真正的目标是什么？（收入、成长、平衡？）
2. 系统思维：当前困境的根本原因是什么？
3. 逆向思维：如何让职业发展失败？
4. 多元视角：心理学（动机）、经济学（机会成本）、职业发展理论

### 案例3：产品方向决策（模板）

**问题**：产品应该往哪个方向发展？

**分析框架**：
1. 第一性原理：用户真正的需求是什么？
2. 系统思维：市场结构和竞争格局
3. 多元视角：用户心理学、商业模式、技术可行性
4. 逆向思维：如何让产品失败？

## 高级技巧

### 技巧1：中途调整

在分析过程中，你可以随时说：
- "够了，直接给建议"（加速）
- "再深入一点"（减速）
- "用逆向思维再看看"（切换工具）
- "从XX视角再分析"（增加视角）

### 技巧2：指定输出格式

- "给我一个简短版本"（只要执行摘要和核心建议）
- "生成完整报告"（默认，8个章节）
- "重点关注行动建议"（侧重实操）

### 技巧3：保存和分享

所有分析报告都会自动保存为 Markdown 文件，你可以：
- 直接在 Markdown 编辑器中查看
- 导出为 PDF 分享
- 复制到笔记软件（Notion、Obsidian）

## 常见问题

### Q1：分析需要多长时间？
- 快速模式：5-10分钟
- 标准模式：20-30分钟
- 深度模式：30-60分钟

### Q2：我可以中途打断吗？
可以！随时说"够了"或"跳过这部分"。

### Q3：分析报告保存在哪里？
默认保存在你的用户目录（`~/`），文件名为"[问题主题]系统性分析报告.md"。

### Q4：我可以修改分析框架吗？
可以！技能文件在 `~/.claude/skills/deep-analysis.md`，你可以编辑它来定制框架。

### Q5：这个技能适合所有问题吗？
不是。简单问题、技术问题、紧急问题不适合用这个技能。它最适合复杂的、需要深度思考的战略性问题。
