# 常见问题解答

## 基础问题

### Q1：分析需要多长时间？
- **快速模式**：5-10分钟
- **标准模式**：20-30分钟（默认）
- **深度模式**：30-60分钟

### Q2：我可以中途打断吗？
可以！随时说"够了"、"跳过这部分"或"直接给建议"。

### Q3：分析报告保存在哪里？
默认保存在你的用户目录（`~/`），文件名为"[问题主题]系统性分析报告.md"。

### Q4：我可以修改分析框架吗？
可以！技能文件在 `01-core-skills/deep-analysis.md`，你可以编辑它来定制框架。

### Q5：这个技能适合所有问题吗？
不是。简单问题、技术问题、紧急问题不适合用这个技能。它最适合复杂的、需要深度思考的战略性问题。

## 使用问题

### Q6：我不知道该用哪个思维工具怎么办？
直接说"帮我深度分析..."，系统会自动选择最合适的工具组合。

### Q7：可以只用一个工具吗？
可以！明确指定即可，比如"只用第一性原理分析..."

### Q8：可以中途切换工具吗？
可以！随时说"用XX工具再看看"或"从XX视角再分析"。

### Q9：如何知道使用了哪些工具？
分析报告中会明确标注使用的工具和分析框架。

### Q10：工具组合有最佳实践吗？
参考 [tool-invocation.md](./tool-invocation.md) 中的"工具选择矩阵"，但不是绝对的，根据具体问题灵活调整。

## 输出问题

### Q11：如何获得更简短的输出？
说"给我一个简短版本"或"只要核心洞察和建议"。

### Q12：如何获得更详细的分析？
说"深度研究"或"需要更多细节"。

### Q13：可以指定输出格式吗？
可以！比如：
- "给我一个简短版本"（只要执行摘要和核心建议）
- "生成完整报告"（默认，8个章节）
- "重点关注行动建议"（侧重实操）

### Q14：报告可以导出吗？
可以！报告是 Markdown 格式，可以：
- 直接在 Markdown 编辑器中查看
- 导出为 PDF 分享
- 复制到笔记软件（Notion、Obsidian）

## 高级问题

### Q15：如何提高分析质量？
1. 提供更多背景信息
2. 明确你的目标和约束
3. 回答澄清问题时尽量具体
4. 要求"挑战我的假设"或"给我反例"

### Q16：如何处理敏感问题？
系统会保护你的隐私，但建议：
- 不要提供真实姓名、公司名等敏感信息
- 可以用"A公司"、"某人"等代称
- 分析报告不会自动分享

### Q17：可以保存分析历史吗？
可以！所有分析报告都是 Markdown 文件，建议：
- 创建一个专门的文件夹保存
- 使用版本控制（Git）管理
- 定期回顾和更新

### Q18：如何与团队分享分析结果？
1. 直接分享 Markdown 文件
2. 导出为 PDF
3. 复制到团队协作工具（Notion、Confluence）
4. 提取核心洞察制作 PPT

## 技术问题

### Q19：需要安装什么软件吗？
不需要！这是一个 Claude 技能，直接在对话中使用。

### Q20：可以在移动设备上使用吗？
可以！只要能访问 Claude，就可以使用这个技能。

### Q21：分析报告的格式是什么？
Markdown 格式，包含：
- 标题和章节
- 表格和列表
- 可选的 Mermaid 图表

### Q22：如何查看 Mermaid 图表？
使用支持 Mermaid 的 Markdown 编辑器，如：
- Typora
- Obsidian
- VS Code（安装 Mermaid 插件）
- GitHub（自动渲染）

## 故障排除

### Q23：分析结果不符合预期怎么办？
1. 提供更多背景信息
2. 明确指出哪里不符合预期
3. 要求"从XX角度再分析"
4. 要求"挑战这个结论"

### Q24：分析过程中卡住了怎么办？
说"跳过这部分"或"继续下一步"。

### Q25：如何重新开始分析？
说"重新开始"或"换一个角度分析"。
