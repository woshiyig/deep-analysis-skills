## 🔷 工具1：第一性原理

### 何时使用

- 遇到"这个行业都这么做"的惯性思维
- 需要创新突破时
- 资源受限需要重新思考时
- 想要找到问题的根本原因

### 如何调用

**直接调用**：
```
"用第一性原理分析为什么我的产品卖不出去"
"帮我用第一性原理拆解这个商业模式"
"从第一性原理出发，分析我们是否应该进入这个市场"
```

**关键词触发**：
- "拆解到基本事实"
- "去除所有假设"
- "从零开始思考"
- "找到根本原因"

### 我会做什么

1. **列出所有假设**
   - 识别你问题中的隐含假设
   - 用表格形式展示每个假设是否成立

2. **质疑每个假设**
   - 提供证据或反证
   - 区分"事实"和"假设"

3. **保留基本事实**
   - 提取不可再分的基础真理
   - 只保留经过验证的事实

4. **重新构建解决方案**
   - 从基本事实出发
   - 构建新的解决路径

### 输出示例

```markdown
## 假设清单
| 假设 | 是否成立 | 证据/反证 |
|------|---------|----------|
| 产品质量不好 | ✗ 未验证 | 需要用户反馈数据 |
| 价格太高 | ✗ 片面假设 | 竞品价格相似 |

## 基本事实
✓ 产品存在
✓ 有用户使用过
✓ 存在负面反馈

## 逻辑推导
事实A + 事实B → 推论1
推论1 + 事实C → 解决方案
```

## 🔷 工具2：系统思维（冰山模型）

### 何时使用

- 问题反复出现
- 治标不治本
- 需要找到根本原因
- 想要理解深层结构

### 如何调用

**直接调用**：
```
"用系统思维分析为什么团队效率低下"
"帮我用冰山模型分析这个问题"
"从系统角度看，为什么我们的改进总是失败"
```

**关键词触发**：
- "找到根本原因"
- "为什么总是这样"
- "深层原因是什么"
- "系统性问题"

### 我会做什么

分析冰山模型的四层：

1. **表象层**：可见的事件
   - 描述你看到的现象

2. **模式层**：重复的模式
   - 识别重复出现的规律

3. **结构层**：系统结构
   - 分析流程、激励、约束

4. **心智模式层**：深层信念
   - 挖掘深层的思维模式和假设

### 输出示例

```markdown
## 冰山模型分析

### 表象层：可见的现象
- 员工经常加班
- 项目总是延期

### 模式层：重复的模式
- 每次项目都低估工作量
- 需求总是在中途变更

### 结构层：系统结构
- 缺乏需求评审流程
- 没有变更管理机制

### 心智模式层：深层信念
- "快速响应客户需求最重要"
- "计划赶不上变化"
```

## 🔷 工具3：逆向思维

### 何时使用

- 风险识别
- 避免失败比追求成功更重要时
- 需要突破思维定式
- 想要找到隐藏的陷阱

### 如何调用

**直接调用**：
```
"用逆向思维分析如何让这个项目失败"
"如果我想搞砸这件事，我会怎么做"
"最坏的情况是什么"
```

**关键词触发**：
- "反过来想"
- "如何避免失败"
- "最坏情况"
- "风险是什么"

### 我会做什么

1. **定义成功状态**
   - 明确你想要的结果

2. **反向定义失败状态**
   - 描述最坏的情况

3. **列出所有导致失败的因素**
   - 识别所有可能的陷阱

4. **逐一避免这些因素**
   - 提供具体的规避策略

### 输出示例

```markdown
## 逆向思维分析

### 正向目标
让产品成功上市

### 反向问题
如何让产品失败？

### 失败因素清单
1. 不做用户调研
2. 忽视竞品分析
3. 定价过高
4. 营销不足
5. 产品质量差

### 规避策略
- 做充分的用户调研 ✓
- 深入分析竞品 ✓
- 制定合理定价策略 ✓
- 投入足够的营销预算 ✓
- 严格质量控制 ✓
```

## 🔷 工具4：多元思维格栅

### 何时使用

- 复杂问题需要多角度验证
- 单一视角陷入死胡同
- 需要创新性解决方案
- 想要全面理解问题

### 如何调用

**直接调用**：
```
"从多元视角分析这个问题"
"用多学科思维分析..."
"从心理学、经济学、管理学角度分析..."
```

**关键词触发**：
- "多角度分析"
- "跨学科思考"
- "全面分析"
- "不同视角"

### 我会做什么

从3-5个学科视角分析问题：

1. **心理学视角**
   - 认知偏差、动机理论、行为改变

2. **经济学视角**
   - 激励机制、机会成本、边际效应

3. **生物学视角**
   - 进化、适应、生态系统

4. **物理学视角**
   - 杠杆、临界点、能量守恒

5. **数学视角**
   - 概率、复利、网络效应

6. **历史学视角**
   - 周期、路径依赖、黑天鹅

### 输出示例

```markdown
## 多元视角分析

### 视角1：心理学
- 自主需求理论：人需要自主感
- 认知失调：行为与信念不一致

### 视角2：经济学
- 激励错配：奖励机制设计不当
- 机会成本：选择A意味着放弃B

### 视角3：管理学
- 领导力风格：指令式 vs 教练式
- 组织文化：信任 vs 控制

### 综合洞察
问题的本质是激励机制和领导风格的不匹配
```

## 🔷 工具5：批判思考（Paul-Elder框架）

### 何时使用

- 做重大决策前
- 评估他人建议时
- 验证自己的分析
- 需要质疑假设时

### 如何调用

**直接调用**：
```
"用批判思考分析这个方案"
"帮我质疑这个假设"
"从批判性角度评估..."
```

**关键词触发**：
- "这个假设成立吗"
- "证据是什么"
- "逻辑严密吗"
- "有没有盲点"

### 我会做什么

从五个维度质疑：

1. **事实（Facts）**
   - 这是事实还是观点？
   - 证据是什么？
   - 数据可靠吗？

2. **证据（Evidence）**
   - 样本量够大吗？
   - 有没有选择性偏差？
   - 来源可信吗？

3. **假设（Assumptions）**
   - 隐含了什么前提？
   - 这些前提成立吗？
   - 如果前提不成立会怎样？

4. **视角（Perspectives）**
   - 其他人会怎么看？
   - 我的盲点是什么？
   - 有没有利益冲突？

5. **推论（Inferences）**
   - 逻辑链条完整吗？
   - 有没有跳跃？
   - 结论是唯一的吗？

### 输出示例

```markdown
## 批判思考分析

### 事实检验
✓ 这是事实：销售额下降20%
✗ 这是观点：产品质量差

### 证据评估
- 样本量：100个用户反馈（偏小）
- 来源：内部调查（可能有偏差）
- 建议：扩大样本，增加第三方调查

### 假设质疑
- 假设1：用户不满意是因为产品质量
  - 反证：可能是价格、服务、竞品等因素
- 假设2：改进产品就能提升销售
  - 反证：可能需要改进营销、渠道等

### 视角拓展
- 用户视角：价格太高
- 销售视角：竞品更有吸引力
- 技术视角：产品功能不足

### 逻辑检验
- 逻辑链：销售下降 → 产品质量差 → 改进产品
- 问题：中间有跳跃，缺少因果证据
```
